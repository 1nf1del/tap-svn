Introduction

Macro is a TAP that executes one or more keys for each macro key defined. Invocation of a macro can be through a single RC key, or a combination of a “function” key together with a RC key (e.g. similar to Alt TAB on a PC). The use of the function key can be varied to be

  • always on, i.e. No function key is required, or
  • use key once, i.e. press the function key once to enable the macros and from then on simply press the RC key to invoke the macro, or
  • use key every time, i.e. you must press the function key with the RC key to invoke the macro

Limitations

NB. this version is BETA quality and has had very limited testing.

Currently only remote control keys are allowed.

Installation

The TAP is intended to run in TSR mode so place it in your Auto Start folder. This can be done using the Topfield Altair tool and a USB cable.

You will also need to copy macro.ini to the same directory as the TAP. Macro.ini holds the macro assignments and any key redefinitions needed for this tap.

Usage

The TAP’s behaviour is modified through options in the file macro.ini. The file should contain up to 3 sections:

·         [macro] – contains key redefinitions for the TAP as a whole (not the macros). For example, the key to stop the TAP, select OK , the function key, etc. Syntax is:

o        <keyword>=<RC key>

For example:

ExitTAP=RKEY_Exit

If you use any FunctionState (see below) other than AlwaysOn, you must have one line that defines your function key as follows:

Function=<RC Key>   where <RC Key> is any remote control key.

·         [options] – contains the options that dictate how the macros will work. Currently there is only one option:

o        FunctionState – this can be one of:

§         AlwaysOn (default) - no need to press a function key to activate macros

§         UseKeyOnce - press the function key once and macros are enabled from then on

§         UseKeyEveryTime - function key must be pressed every time a macro is required

·         [function keys] – contains one line for each macro assignment. Syntax is:

o        <key to be redefined>=<macro list>

o        <key to be redefined> is any one RC key defined in KEY.H (included), e.g. RKEY_Sleep.

o        <macro list> is one or more RC keys defined in KEY.H separate by commas.

For example to redefine the Sleep key to record add the following line:

RKEY_Sleep=RKEY_Record

To redefine the Sleep key to go up 2 channels add the following line:

RKEY_Sleep=RKEY_ChUp,RKEY_ChUp
