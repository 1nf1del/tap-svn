TAP Project - Project setup

Once your Cygwin or other UNIX like environment is set up for TAP Project development, it is time to setup your TAP Project workspace. This document is a step-by-step guide for this initial project setup.
  1. Optional, but highly recommended: If you have not already done so, subscribe to the project mailing lists:
    TAP-announce for project related announcements. This is a moderated, low traffic list.
    TAP-devel for developer talk. Subscribers can post to this list to collaborate.
  2. Get a shell prompt. Cygwin users can do so by double-clicking on the "TAP shell" icon they created as part of their Cygwin setup. You should see something like this:
    username@computer ~
$ _
    This is your shell prompt. It indicates your user name, the computer name and the current directory (~ means your home directory), the $ sign means that the shell is ready to accept input and _ is the cursor.
  3. Once you have your shell prompt, create a working directory for your projects: 
    username@computer ~
$ mkdir -p projects/tap
  4. Enter the projects directory: 
    username@computer ~
$ cd projects/tap
  5. Get a copy of the most up to date TAP project source code: 
    username@computer ~/projects/tap
$ svn checkout http://svn.berlios.de/svnroot/repos/tap/trunk
...
    This may take a little while to download, please be patient.
  6. Enter the tap directory: 
    username@computer ~/projects/tap
$ cd trunk/taps
  7. You can now view the list of the files in the TAP part of the repository. Each TAP has it's own directory and there is a Makefile that will build all TAPs that use the provided build system. 
    username@computer ~/projects/tap/trunk/taps
$ ls
DumpAPI ViewMaster3 gcode skeleton usbmon
Makefile freespacebar macro timer_extend
  8. To start your own TAP project: 
    username@computer ~/projects/tap/trunk/taps
$ mkdir my_own_tap
username@computer ~/projects/tap/trunk/taps
$ cp skeleton/* my_own_tap
username@computer ~/projects/tap/trunk/taps
$ cd my_own_tap
username@computer ~/projects/tap/trunk/taps/my_own_tap
$ mv my.c my_own_tap.c
    Then edit Makefile and update the name of the TAP. The comments in the Makefile explain a few other things you may wish to do.
  9. Compile your TAP: 
    username@computer ~/projects/tap/trunk/taps/my_own_tap
$ make
  10. (optional) Remove (clean out) all compiled files, including the TAP: 
    username@computer ~/projects/tap/trunk/taps/my_own_tap
$ make clean
    You will only want to do this rarely, for example when making a source code release and you want to remove the generated files.
  11. You can update to the latest version of the project source code in any directory by issuing the Subversion update command: 
    username@computer ~/projects/tap/trunk/taps/my_own_tap
$ cd ../.. 
username@computer ~/projects/tap/trunk
$ svn update
...
  12. (Optional) You can install a GUI Subversion client if you don't feel comfortable with the command line version. However, it is still recommended that you use the command line version for this initial project setup.
